## Resumo das mudanças
Foi criado um novo arquivo `agent.py` que implementa uma ferramenta para gerar um resumo técnico de um diff git entre a branch atual e a branch `main`. O script executa comandos git para obter o diff, limita o tamanho do diff para evitar uso excessivo de tokens, carrega variáveis de ambiente com `dotenv`, e utiliza a API OpenAI para enviar o diff e receber um resumo técnico. Também foi ajustado o arquivo `main.py` para carregar a variável de ambiente com `load_dotenv()` antes de instanciar o cliente OpenAI. Por fim, foi adicionado um arquivo `requirements.txt` com as dependências `openai`, `python-dotenv` e `pydantic`.

## Riscos e possíveis bugs
- A função `run_git` pode lançar exceção se o comando git falhar, o que não é tratado no `main()`, podendo causar término abrupto do script.
- O script assume que está sendo executado dentro de um repositório git válido, mas a verificação está comentada.
- O modelo `gpt-4.1-mini-2025-04-14` pode não estar disponível ou pode ser um nome fictício, o que causaria erro na chamada da API.
- O limite de 50.000 caracteres para o diff pode truncar mudanças importantes sem aviso detalhado.
- A chave da API OpenAI deve estar configurada corretamente no ambiente, caso contrário a chamada falhará.

## Testes recomendados
- Testar execução do script em um repositório git válido com mudanças pendentes para verificar se o resumo é gerado corretamente.
- Testar execução em diretório que não seja repositório git para validar tratamento de erro.
- Testar com diffs muito grandes para verificar truncamento e comportamento do script.
- Validar se a variável de ambiente da API OpenAI está sendo carregada corretamente e se a autenticação funciona.
- Testar integração com a API OpenAI para garantir que o modelo e o formato da requisição estão corretos.
